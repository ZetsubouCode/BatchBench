Flatten & Renumber

Copy or move images from one folder tree into a single numbered output folder.
Example: 001.png, 002_1.png
Typical workflow
1. Pick RootChoose the chapter or dataset folder that contains loose images and/or image subfolders.
2. PreviewKeep Dry run on first to review the planned names in the log.
3. RunUse Copy for a safe new folder, or Move when you intentionally want to empty the source.
Source & Output
Top-level images become single numbers. Images inside each subfolder share one prefix with part numbers.
Leave empty to write into _renamed inside the root.
Case-insensitive. Example: .png,.jpg
Numbering
e.g. 001
e.g. _01 vs _1
Between prefix and part: 002_1
Live example
001.png, 002_1.png, 002_2.png
Ordering
Name (natural) is usually best for manga/webtoon pages because 2.png sorts before 10.png.
Options
The log shows the first 20 planned renames before writing.
