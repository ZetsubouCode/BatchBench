Workflow Guide

How to use each menu correctly

BatchBench is a dataset preparation toolbox. The easiest way to avoid mistakes is to pick the right tab for the current step, keep source and output folders separate, and use preview-style actions before writing files.

Folder roles

Source reads your current files. Temp is a staging area. Output writes new results. Working is Pipeline's scratch space.

Preview first

Use scan, dry-run, preview, or a small limit before applying a change to the full dataset.

Keep backups on

Backups cost disk space, but they are the simplest recovery path when a preset or tag rule is wrong.

Menu explanations

Open the section for the tool you want to use. Each dropdown explains when to use it, the normal steps, and the most common source of confusion.
